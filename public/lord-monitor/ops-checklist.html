production checklist

LORD monitor publish and bridge verification

Use this page after the site republish to walk the remaining path without guessing.

1. Open the monitor entry
Load /lord-monitor/ and confirm the control center, display, audio, and 3D pages are reachable.
2. Verify browser websocket leg
Open bridge-test.html, start the Python bridge listener locally, then send the sample fusion-update payload.
3. Verify Python listener
Run python -m src.integrations.lord_bridge.lord_bridge or the project’s chosen runtime entrypoint, then execute smoke_test.py to send a known-good payload.
4. Confirm event flow
Check that the bridge logs the payload, computes urgency, and materializes normalized bridge state.
5. Close the loop
Once verified, mark the related Airtable records complete and update the Confluence runbook blocker note.
